Что такое REST?

Объяснение архитектурного стиля для построения веб-API

Определение REST

REST (REpresentational State Transfer) — это архитектурный стиль для проектирования сетевых приложений. Он определяет набор ограничений, которым должно следовать API, чтобы быть "RESTful".

Основные принципы REST

  • Без сохранения состояния (Stateless): каждый запрос должен содержать всю необходимую информацию.
  • Единообразие интерфейса: простые и понятные URL, стандартные методы HTTP.
  • Разделение на клиент и сервер: клиент и сервер независимы друг от друга.
  • Кэширование: ответы могут быть кэшируемыми.
  • Многоуровневая система: клиент не знает, подключён ли он напрямую к серверу или через промежуточный слой.

Пример REST API

Допустим, у нас есть API для управления пользователями: 

GET    /api/users       → Получить список пользователей
GET    /api/users/1     → Получить пользователя с ID=1
POST   /api/users       → Создать нового пользователя
PUT    /api/users/1     → Обновить пользователя с ID=1
DELETE /api/users/1     → Удалить пользователя с ID=1

HTTP-методы в REST

REST активно использует стандартные HTTP-методы:

  • GET — получение данных
  • POST — создание ресурса
  • PUT — обновление существующего ресурса
  • PATCH — частичное обновление
  • DELETE — удаление

Форматы данных

REST-сервисы чаще всего работают с данными в форматах:

  • JSON — наиболее популярный формат
  • XML — используется реже, но всё ещё встречается

REST vs SOAP

REST SOAP
Легковесный, основан на HTTP Тяжёлый, имеет собственный протокол
Использует JSON/XML Использует XML
Прост в разработке и тестировании Сложнее из-за строгих спецификаций
Широко используется в веб- и мобильных приложениях Часто применяется в корпоративных системах

REST vs GraphQL

GraphQL — альтернатива REST, позволяющая клиенту запрашивать именно те данные, которые ему нужны.

REST GraphQL
Множество эндпоинтов Один эндпоинт
Нужно несколько запросов для получения разных данных Все данные можно получить за один запрос
Подходит для простых API Полезен при сложных связях между данными

Заключение

REST — это гибкий и удобный способ создания веб-API, основанный на стандартах HTTP. Благодаря своей простоте и широкому распространению, REST стал стандартом де-факто для большинства современных веб-приложений.
